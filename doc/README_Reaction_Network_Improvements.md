# ReaxTools 反应网络算法改进文档集

## 文档概述

本文档集针对 ReaxTools 中反应网络分析模块的现存问题，提出系统性的算法改进方案。

### 现存问题

1. **反应过于繁杂** - 燃烧等复杂体系产生数百个反应，难以分析
2. **主次不分** - 无法区分主干反应路径与次要/瞬态反应
3. **可视化困难** - 即使简化的网络图仍然过于密集
4. **缺乏深度分析** - 仅统计频次，缺乏网络拓扑和时序分析

## 核心算法对比

### 反应重要性评估：旧 vs 新

**旧方法**（仅基于频次）：
```cpp
// 只考虑反应次数
std::sort(edges.begin(), edges.end(), 
    [](a, b) { return a.count > b.count; });
output_top_n(edges, n=60);
```

**新方法**（多层次综合评估）：
```cpp
// 层次1: 基础统计
base_score = 0.3*freq + 0.2*atom_economy + 0.1*time_span

// 层次2: 网络拓扑
topo_score = 0.15*betweenness + 0.1*pagerank

// 层次3: 化学合理性
chem_score = 0.1*stability + 0.05*reaction_type_bonus

// 综合
importance = base_score + topo_score + chem_score
```

### 网络简化：旧 vs 新

**旧方法**（简单截断）：
```cpp
// 只保留前60个高频反应
if (edges.size() > 60) {
    edges.resize(60);
}
```

**新方法**（智能聚合）：
```cpp
// 步骤1: 过滤低质量反应
filter_by_atom_economy(threshold=0.3);

// 步骤2: 社区发现分组
communities = detect_communities(resolution=1.0);

// 步骤3: 聚合社区为超节点
write_community_aggregated_dot(communities);

// 步骤4: 社区内保留关键边
for each community:
    keep_top_betweenness_edges(n=5);
```

## 关键技术决策

### 决策1：有向图 vs 无向图
**选择**：保持有向图
- 理由：反应方向对化学机理至关重要
- 影响：社区发现和中心性算法需要适配有向图

### 决策2：精确算法 vs 近似算法
**选择**：介数中心性使用采样近似
- 大规模系统（>1000节点）：10%采样
- 中小规模系统（<1000节点）：精确计算
- 理由：近似结果足够好，速度提升10倍

### 决策3：静态分析 vs 动态分析
**选择**：两者结合
- 静态：社区结构、瓶颈识别
- 动态：时序验证、爆发检测
- 输出：静态网络+动态标注
